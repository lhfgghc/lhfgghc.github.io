Vue.js

1.概要介绍

概念

Vue是一套用于构建用户界面的渐进式框架，Vue被设计为自底向上逐层应用。

声明式编程：不需要直接操作DOM元素。Vue是一个响应式系统：数据和DOM元素绑定在一起，所有的元素都是响应式的。

MVVM模式

Model-View-ViewModel。Model指的是数据部分，主要负责业务数据；View指的是视图部分，即DOM元素，负责视图的处理；ViewModel连接视图和数据的数据模型，负责Model监听或者View的修改。

把需要改变的视图数据初始化到Vue中，然后通过修改Vue中的数据从而实现对视图的更新。

核心技术

组件化和虚拟DOM技术。Vue通过虚拟DOM技术来减少对DOM的直接操作。

2.基础技术

2.1 基本架构

<body>
    <div id="app">
        <p>{{msg}}</p>  <!-- 插值表达式，将data中数据渲染到页面 -->
        <input type="text" v-model="msg">  <!-- 数据双向绑定 -->
    </div>
    <script>
        // 实例化Vue程序
        var vm = new Vue({
            el:'#app',  //Vue管理的DOM节点入口，值为选择器 不能为html或者body
            data: {     //指定初始化数据
                msg:"Hellowrold"
            }
        }) 
    </script>    
</body>

2.2 Vue实例

el唯一根标签

class或者id选择器可以用来将页面结构与Vue实例对象中的el绑定。

data初始数据

Vue的数据对象为data，Vue会将data的数据转换为get、set接口从而让data的属性能够相应数据变化。

可以通过vm.$data访问原始数据对象。vm.name等价于vm.$data.name，由于Vim实例代理了data对象的所有属性。

methods定义方法

methods属性用来定义方法，方法中this指向Vue实例本身。

其中绑定事件语句**@click等价于v-on:click**，v-on为事件监听指令，直接与事件类型配合使用。

<button @click="showInfo">Click</button>  <!-- 绑定事件 -->

var vm = new Vue({
    el:'#app',  
    data: {
        msg:"Hellowrold"
    },
    methods: {
        //定义事件处理方法
        showInfo() {
            this.msg="Click"  //this指向该Vue实例vm
        }
        //或者写为
        showInfo:function() {
    		
		}
    }
})
computed计算属性

一些数据需要随着其他数据的变动而变动，使用computed计算属性。

事件处理方法中，this指向的Vue实例的计算属性结果会被缓存起来，只有依赖的响应式属性变化时，才会重新计算，返回最终结果。

<body>
    <div id="app">
        <p>自变量X:{{dataX}}</p>
        <input type="text" v-model="dataX">  //双向绑定数据
        <p>自变量Y:{{dataY}}</p>
        <input type="text" v-model="dataY">	 //双向绑定数据
        <p>计算结果X+Y:{{Sum}}</p>
    </div>
    <script>
        var vm = new Vue({
            el:"#app",  //挂载唯一根标签
            data: {
                dataX:0,
                dataY:0,
            },
            computed: {
                //计算属性 Sum作为绑定的目标数据
                Sum() {
                    return Number(this.dataX) + Number(this.dataY)
                }
            }
        })
    </script>
</body>
watch状态监听

原有的事件出发不能自动监听Vue实例的状态变化，watch只需监听当前Vue实例中的数据变化。

例如：

watch: {
	Sum(newName,oldName) {
		console.log(newName,oldName)
	}
}

监听Sum变量的变化，如果发生变化则打印旧值和新值。

filter过滤器

可以在插值表达式中使用filter，使用管道符|，语法为**{{data|filter}}**。

其中filter定义为函数形式，将data作为参数传入filter函数，以实现指定操作。P9例代码

v-bind属性绑定也可以使用过滤器，如“v-bind:id=“data”，表示绑定id属性为data，data后面可以加过滤器。

2.3 Vue数据绑定

绑定样式

使用v-bind将样式数据绑定给DOM元素。

可以1.绑定样式属性；2.绑定样式对象；3.绑定样式类

<body>
    <div id="app">
        <div v-bind:style="{backgroundColor:pink,width:width,height:height}">
            这里使用的是绑定样式属性值，具体值也是Vue实例的data数据
        </div>
        <div v-bind:style="myDiv"> 
            这里使用的是绑定样式对象，对象在Vue实例data中定义
        </div>
        <div v-bind:class="box">
            这里使用绑定样式类class的方法实现样式的绑定，类值也在Vue实例data中
        </div>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data: {
                //样式对象
                myDiv:{backgroundColor:'red',width:'300px',height:'100px'},
                //样式属性值
                pink:'pink',
                width:'100%',
                height:'200px',
                box:'box' //box类的样式在style标签中声明
            }
        })
    </script>
</body>
v-model指令

实现数据的双向绑定，通常用于表单数据上，格式为：v-model=“data”

v-text指令

实现在DOM元素内部插入文本内容，格式为：v-text=“txt”

v-html指令

在DOM元素内部插入HTML标签内容，可以是HTML结构。

v-bind指令

实现属性数据的单向绑定，格式为：v-bind:value=“values”。其中value为元素属性，如style、value等，可以简写为： :value=“values”。

v-on指令

属于事件监听指令，直接与事件类型配合使用。格式为：v-on:events=“methods”，其中events为事件类型，如click，methods为Vue实例中定义的具体方法。可以简写为@events=”methods“。

注意，event会作为响应函数的默认形参，代表原生DOM事件，当调用的函数有多个参数传入时，需要使用原生DOM事件时，则通过$event作为实参传入。

v-for指令

遍历数组，以**（项，索引）**的形式遍历，其中索引可以省略，但高版本的Vue最好不要省略。

迭代对象时，语法为v=for=”(value,key,index) in Object"。其中value为每个对象的属性值，key为属性名，index为索引值。后两个可选。

<body>
    <div id="app">
        <!-- 以（项，索引）的形式遍历data列表 -->
        <div v-for="(item,index) in list">
            索引为:{{index}},内容为:{{item}}
        </div>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data: {
                list:[2,3,4,5]
            }
        })
    </script>
</body>
v-if指令

控制元素的隐藏和显示，和v-show功能类似，v-show操作的是display属性，元素始终被渲染并保留在DOM中，而v-if会对元素进行删除和重建，性能较差。

<body>
    <div id="app">
        <!-- 绑定事件，反转flag -->
        <button @click="flag=!flag">Click</button>
        <!-- 判定flag，为true则显示 -->
        <div v-if="flag" style="background-color:red">Try v-if</div>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data:{
                flag:true
            }
        })
    </script>
</body>

2.4 Vue事件

使用按键修饰符监听按键

监听键盘事件时，需要检查常见的键值。Vue允许为v-on添加按键修饰符来监听按键。

例如v-on:keyup.enter=“func”。该事件为回车键被按下。

格式：v-on:keyup.按键名，或者@keyup.按键名

常见的按键名包括.enter .tab .delete .esc .space …

.stop阻止事件冒泡

通过在事件后面加.stop,例如click.stop，可以使得事件冒泡行为被阻止，即事件不会向上传递至父级元素。

.prevent阻止默认事件行为

对于HTML标签的自身特性，例如a会进行自动跳转，可以使用.prevent修饰符对事件进行控制，使得元素的默认事件行为被阻止。例如在a标签中加入语句v-on:click.prevent，即可阻止跳转行为。

.capture事件捕获

对于默认情况，事件由内到外执行，即冒泡顺序，而.capture事件捕获则时由外及内的。

.self自身触发

该事件修饰符来实现只有DOM元素本身会触发事件。即若某事件是冒泡触发的，而非其元素本身受到的触发，则不会执行事件触发。

.once只触发一次

阻止事件的多次触发。

修饰符顺序

事件修饰符可以控制事件按照一定规则触发。

例如，v-on:click.prevent.self阻止所有的单击,v-on:click.self.prevent阻止对元素本身的单击。

2.5 Vue组件

全局注册

使用Vue.component()方法进行全局注册组件。

1.通过Vue.extend()方法生成组件构造器，内容包含data，template，watch等。(与new Vue接收相同的选项)

（Tips:此类配置选项在组件内部有独立作用域，不能从外面直接访问。）

2.调用Vue.component()方法注册组件。

3.在Vue实例的作用范围使用组件。

例如：

var myConponent = Vue.extend({
    //data在组件中必须为一个函数，通过返回值来返回初始数据
    data() {
        return {
            count:0
        }
    },
    template:'<button v-on:click="count++">被单击{{count}}次</button>'
})
//2.注册组件
Vue.component('my-conponent',myConponent);
//3.挂载app并在元素中使用组件
var vm = new Vue({
    el:'#app'
})

其中组件构造器可以省略调用extend方法构造，用JavaScript对象来定义，如下。

Vue.component('myConponent',{
    //data在组件中必须为一个函数，通过返回值来返回初始数据
    data() {  
        return {
            count:0
        }
    },
    template:'<button v-on:click="count++">被单击{{count}}次</button>'
})
局部注册组件

通过Vue实例的components属性来实现，如下。其中myConponent1为组件构造器。

var vm = new Vue({
	el:'#app',
	components:{ myConponent: myConponent1}
})
template模板

Vue提供了<template>标签来定义结构的模板，在该标签中书写HTML代码，通过id值绑定到目标组件内的template上。

例如<template id=“tmp1”> HTML代码 <\template>，组件构造器内：template:’#tmp1'

组件间切换

可以采用v-if和v-else来进行显示与否的切换。也可以使用<component>标签的is属性值来进行组件的切换，is的属性值即为显示的组件名称。通过v-bind对该属性值进行绑定即可实现切换。

<body>
    <div id="app">
        <!-- 超链接事件被取消默认响应，事件操作为修改comName -->
        <a href="#" @click.prevent="comName='login'">Login</a>
        <a href="#" @click.prevent="comName='register'">Register</a>
        <!-- 通过component标签的is属性进行显示的组件切换，v-bind将该属性进行绑定 -->
        <component v-bind:is="comName"></component>
    </div>
    <script>
        // 组件定义
        Vue.component('login',{
            template:'<div>登录页面</div>'
        })
        Vue.component('register',{
            template:'<div>注册页面</div>'
        })
        var vm = new Vue({
            el:'#app',
            data:{
                comName:'login'
            }
        })
    </script>
</body>

2.6 组件间通信

组件实例具有局部作用域，需要一些工具来实现组件间通信，例如父组件向子组件传递数据信息。

props传值

props即为道具，用来接受父组件中定义的数据，其值为数组，数组中是父组件传递的信息。

props是以从上到下的单向数据流传递，且父级组件的props更新会向下流动到子组件，但反过来不行。

props在声明组件对象中（组件构造器）定义，有三种方式：（1）数组形式传递属性名；（2）对象形式传递属性名和数据类型；（3）指定属性名、数据类型、必要性、默认值。

举例：

<body>
    <template id="tmp1">
        <!-- 采用template标签来构造组件模板 -->
        <div>父组件消息:{{name}}</div>
    </template>
    <div id="app">
        <!-- 父组件处传入name的值 -->
        <my-parent name="传入信息12132"> </my-parent>
    </div>
    <script>
        Vue.component('my-parent',{
            //在组件构造器中声明，表示接受name数据，它在父组件处传入
            props:['name'],
            template:'#tmp1'
        })
        var vm = new Vue({
            //挂载
            el:'#app'
        })
    </script>
</body>

Tips：如果在默认情况下，无法从外面去给组件内部的值进行赋值，使用props可以想象成开通某个管道，使得数据能够通过该管道进入到组件内部。

可以使用v-bind来动态绑定传入数据的值，如下代码。

<body>
    <template id="tmp1">
        <!-- 采用template标签来构造组件模板 -->
        <div>父组件消息:{{name}}</div>
    </template>
    <div id="app">
        <!-- 使用v-bind来绑定name这个属性，值也是从父节点（此处）来的 -->
        <my-parent v-bind:name="inputdata"> </my-parent>
        <!-- 双向绑定 -->
        <input type="text" v-model="inputdata">
    </div>
    <script>
        Vue.component('my-parent',{
            //在组件构造器中声明，表示接受name数据，它在父组件处传入
            props:['name'],
            template:'#tmp1'
        })
        var vm = new Vue({
            //挂载
            el:'#app',
            data:{
                //声明一个数据，从来接受文本框的数据
                inputdata:0
            }
        })
    </script>
</body>
$emit传值

$emit能够将子组件中的值传递到父组件中，触发父组件中定义的事件，子组件的数据信息通过传递参数的方式完成。

例如：子组件通过this.$emit触发父组件的自定义函数，将其data作为参数传出，父组件通过事件响应函数接收该传出的参数，完成组件间传值。

<body>
    <div id="app">
        <parent></parent>
    </div>
    <template id="child">
        <div>
            <button @click="click"> Send </button>
            <!-- 双向绑定给子组件的data -->
            <input type="text" v-model="message">
        </div>
    </template>
    <template id="parents">
        <div>
            <!-- 在child组件上绑定了一个名为childfn的事件，事件处理方法为transContent -->
            <child @childfn="transContent"></child> From child: {{message}}
        </div>
    </template>
    <script>
        //父组件parent
        Vue.component('parent',{
            template:'#parents',
            data() {
                return {
                    message:''
                }
            },
            methods:{
                //对于事件处理方法，将传入的形参赋值给父组件的数据message
                transContent(payload) {
                    this.message = payload
                }
            }
        })
        //子组件child
        Vue.component('child',{
            template:'#child',
            data() {
                return{
                    message:'child message'
                }
            },
            methods: {
                click() {
                    //按钮按下后响应该方法
                    //子组件触发自定义事件childfn，并且将message作为参数传出
                    this.$emit('childfn',this.message);
                }
            }
        })
        var vm = new Vue({el:'#app'})
    </script>
</body>

2.7 Vue的生命周期

每个Vue实例在被创建时要经过一系列初始化过程，三大阶段：初始化显示、更新显示、销毁Vue实例

初始化阶段的钩子函数：

beforeCreate()实例创建前：数据和模板均为获取到

created()实例创建后：最早可访问到data数据，但模板未获取到

beforeMount()数据挂载前：模板已经获取到，但数据未挂载到模板上

mounted()数据挂载后：数据已经挂载到模板中

更新阶段的钩子函数：

beforeUpdate()模板更新前：data改变后，更新数据模板之前调用

updated()模板更新后：将data渲染到数据模板中

销毁阶段的钩子函数

beforeDestroy()实例销毁前

destroyed()实例销毁后，vm实例在这两个阶段都存在，但是销毁之后获取不到页面中的div，所以，实例销毁后无法操作DOM元素。

2.8 自定义指令

除了内置指令外，Vue允许注册自定义指令。

注册全局指令
//指令名不要带v-
Vue.directive('指令名',{
    //el代表使用该指令的DOM元素
    //binding可获取使用了此指令的绑定值
    inserted:function(el,binding) {
        //逻辑代码
    }
})
注册局部指令
directive:{
	'指令名':{
        inserted:function(el,binding) {
            //逻辑代码
        }
    }
}
全局自定义的案例
Vue.directive('upper-text',{
    //一般对样式的操作在bind中
    bind:function(el) {
        el.style.color='red'
    },
    //一般对js的操作在inserted中
    //el为指令作用的DOM元素
    //binding用于获取使用该指令的绑定值(value)、表达式(expression)
    inserted:function(el,binding) {
        el.innerHTML=binding.value.toUpperCase()
    }
})

2.9 一些全局API

Vue.use（插件相关）

用于在Vue中安装插件，通过插件可以为Vue添加全局功能。插件可以是一个对象或者函数，如果是对象，必须提供install方法，用来安装插件。

使用Vue.use进行插件的安装，随后可以使用插件中的内容。

<body>
    <!-- 使用插件中的自定义指令 -->
    <div id="app" v-my-directive> </div>
    <script>
        // 定义一个对象，表示插件
        let MyPlugin = {}
        // 定义该插件的安装函数，传参Vue构造器，和可选的配置对象
        MyPlugin.install = function (Vue, options) {
            console.log(options)
            //插件中注册全局的自定义指令
            Vue.directive('my-directive',{
                bind(el,binding) {
                    el.style='width:100px;height:100px;background-color:#ccc'
                }
            })
        }

        //安装插件
        Vue.use(MyPlugin,{someOption:true})
        //实例化Vue对象并挂载元素
        var vm = new Vue({
            el:'#app'
        })
    </script>
</body>
Vue.extend（构造子类）

用来构建一个Vue子类，对Vue构造器进行拓展。

<body>
    <!-- app1报错，Vue中没有title，app2则不报错，Vue2是Vue的子类，包含title的data -->
    <!-- <div id="app1">app1:{{title}}</div> -->
    <div id="app2">app2:{{title}}</div>
    <script>
        var Vue2 = Vue.extend({
            data() {
                return { title:'hello' }
            }
        })
        // var vm1 = new Vue({
        //     el:'#app1'
        // })
        var vm2 = new Vue2({
            el:'#app2'
        })
    </script>
</body>
Vue.set（添加属性）

用于向响应式对象中添加一个属性，确保新属性也是响应式的。格式为Vue.set(vm.obj,’tar’,‘data’)。及向Vue实例vm中的obj数据对象中添加一个属性tar，属性值为data。需注意无法动态地添加根级属性，即添加的属性不能和obj同级。

Vue.mixin（全局混入）

用于全局注册一个混入（Mixin），将影响之后创建的每一个Vue实例。向实例中注入指定的行为（函数）。

<body>
    <div id="app"> </div>
    <script>
        // 注册全局混入
        Vue.mixin({
            //注入钩子函数created
            created() {
                //取到自定义属性myOption并在控制台输出
                var myOption = this.$options.myOption
                if(myOption) {
                    console.log(myOption.toUpperCase())
                }
            }
        })
        //实例化时，由于全局注入了，钩子函数生效
        var vm = new Vue({
            //钩子函数检测该属性并在控制台输出
            myOption:"Hello Mixin!"
        })
    </script>
</body>

对于混入，局部创建则为在Vue实例化时为mixins属性传入一个混入对象。

//混入对象
var mixin = {
    created: function () {
        document.write('混入调用' + '<br>')
    }
}
//传入混入对象，以数组形式传入，这样会进行合理的合并
new Vue({
    mixins: [mixin],
    created: function () {
        document.write('组件调用' + '<br>')
    }
});

如果 methods 选项中有相同的函数名，则 Vue 实例优先级会较高。其次是混入对象中的。

vm.$options

2.11 createElement&render 渲染

createElement()函数返回的是一个描述节点，用来告诉Vue在页面上渲染什么样的节点，亦称为虚拟节点，VNode。

第一个参数：HTML标签名或者组件选项对象

第二个参数：可选，可以传入一个与模板中属性相对应的数据对象

第三个参数：是由createElement构建而成的子级虚拟节点，也可以用字符串生成文本虚拟节点。

代码样例1：含有默认插槽

<body>
    <div id="app">
        <my-component> 成功渲染，此处为默认插槽 </my-component>
    </div>
    <script>
        // 注册组件
        Vue.component('my-component',{
            render(createElement) {
                // 传入createElement函数返回的VNode
                // 该VNode告知标签类型、样式属性、子级虚拟节点
                return createElement('p',{
                    style: {
                        color:'red',
                        fontSize:'16px',
                        backgroundColor:'#eee'
                    }
                    //this.$slots.default 理解为一个创建默认插槽
                },this.$slots.default)
            }
        })
        var vm = new Vue({el:'#app'})
    </script>
</body>

代码样例2：含有命名插槽

<body>
    <div id="app">
        <my-component>
            <!-- 各模板通过v-slot来定义指定的命名插槽 -->
            <template v-slot:header>
                <div style="background-color: #ccc;height: 50px;">
                    header
                </div>
            </template>

            <template v-slot:content>
                <div style="background-color: #ddd;height: 50px;">
                    content
                </div>
            </template>

            <template v-slot:footer>
                <div style="background-color: #eee;height: 50px;">
                    footer
                </div>
            </template>
        </my-component>
    </div>
    <script>
        Vue.component('my-component',{
            render(createElement) {
                //该VNode类型为div，包含三个子级虚拟节点
                return createElement('div',
                    [
                        //三个子级虚拟节点，其各自的子节点包含命名插槽
                        createElement('header',this.$slots.header),
                        createElement('content',this.$slots.content),
                        createElement('footer',this.$slots.footer),
                    ]
                )
            }
        })
        var vm = new Vue({el:'#app'})
    </script>
</body>
render函数写法

render函数由Vue调用，其中传入的参数为createElement函数，返回参数为createElement函数的返回值。

render(createElement) {
	return createElement(...)
}
render:(createElement) => {return createElement(...)}
render:createElement =>createElement(...)
render:h=>h(APP)//h传入组件

3.Vue过渡和动画

3.1 transition组件

Vue为<transition>标签内部的元素提供了三个进入过渡和三个离开过渡的类。

内置的class类名实现过渡
<body>
    <div id="app">
        <!-- 用于div的隐藏和显示的切换 -->
        <button v-on:click="toggle"> Click </button>
        <!-- name属性设置在过渡中切换的类名前缀,默认为v- -->
        <transition name="box">
            <div class="chart" v-if="show"> </div>
        </transition>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data: {
                show:true,
            },
            methods: {
                toggle() {
                    this.show=!this.show
                }
            }
        })  
    </script>
</body>

其中transition标签设置name，前缀为box，在CSS中实现以该box-为前缀的过渡类，不设置则默认为v-。该标签自动应用这些class在达到效果。

过渡类有：v-enter、v-enter-active、v-enter-to、v-leave、v-leave-active、v-leave-to。

在CSS中操作transition（过渡）和animation（动画）来达到不同的效果。

.chart {
	width: 200px;
	height: 50px;
	background-color: aqua;
}
/* 通过css内置类名来实现过渡,name前缀为box */
.box-enter-active, .box-leave-active {
	transition: width 3s; /*某一项属性 + 持续的时间*/  
}
.box-enter, .box-leave-to {
	width: 0px;
}
.box-enter-to, .box-leave {
	width: 200px;
}
自定义类名

无需对transition标签的name属性进行设置，可以通过设置以下属性来配置自定义的css类。例如animate.css中已定义了一系列动画类。

enter-class、enter-active-class、enter-to-class、leave-class、leave-active-class、leave-to-class

appear初始渲染动画

给元素添加初始渲染的动画效果。在transition标签中进行相关属性的设置。

appear-class表示初始class样式；appear-to-class表示过渡完成的class样式；appear-active-class会应用到整个过渡过程中。

三者的顺序要求见vue.js教材P69。

<div id="app">
	<button @click="show=!show"> Click </button>
	<!-- 为transition标签设置appear属性
		第一个appear代表开启此特性 -->
	<transition appear appear-active-class="animated swing"
			enter-active-class="animated bounceIn"
			leave-active-class="animated bounceOut">
		<div v-if="show"> Hello </div>
	</transition>
</div>
使用@keyframes创建CSS动画

与前面提到的CSS过渡用法相比，@keyframes动画中的v-enter类名在节点插入DOM后并不会立即删除，而是在animationend（动画结束）事件触发时删除。

@keyframes规则创建动画，即将一套CSS样式逐步演变成另一套，过程中可以多次改变CSS样式，通过百分比或者from和to（相当于0%和100%）来规定动画的状态。

基本语法如下为： @keyframes 规则名字 {百分比 {样式；}}

例子如下：

<body>
    <div id="app">
        <button @click="show=!show">@keyframes创建CSS动画</button>
        <!-- 内置class类实现过渡的方法 -->
        <transition name="bounce">
            <div class="circular" v-if="show">原型</div>
        </transition>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data:{
                show:true
            }
        })
    </script>
</body>

transition标签中绑定name属性为bounce，在CSS中的过渡类将以他为前缀。CSS代码如下。

<style>
    div.circular {
        width: 100px;
        height: 100px;
        background: red;
        border-radius: 50%;
        margin-top: 20px;
        text-align: center;
        line-height: 100px;
        color:aqua;
    }
    /* CSS中操作动画,通过animation */
    .bounce-enter-active {
        animation: Ami .5s;
    }
    .bounce-leave-active {
        animation: Ami .5s;
    }
	/* 定义动画规则 */
    @keyframes Ami {
        0% {
            transform:scale(0);
            background: aqua;
        }
        20% {
            transform:scale(1);
            background: burlywood;
        }
        50% {
            transform:scale(1.5);
            background: blueviolet;
        }
        100% {
            transform:scale(1);
            background: burlywood;
        }
    }
</style>
常见的一些CSS属性

animation：绑定动画，后带上绑定的动画规则名+时长

transition：绑定过渡，后带上需要变动的属性+时长，如果属性为all则为所有效果

opacity：不透明度，0为隐藏，1为显示

transfrom：包括scale缩放，translateX水平方向移动等。

使用钩子函数实现动画

Vue可以借助JavaScript来完成动画。在<transition>标签中定义了一些钩子函数。在methods中编写钩子函数。示例见教材P71。

3.2多元素过渡

不同标签名的元素过渡

不相同标签名的元素可以使用v-if和v-else来实现过渡。

相同标签名元素的过渡

需要通过key特性设置唯一值来标记，通过key让Vue来进行区别。

<body>
    <div id="app">
        <!-- 进行切换 -->
        <button @click="f=!f">Click</button>
        <div>
            <transition name="fade">
                <!-- 设置唯一的key值以区分button -->
                <button v-if="f" key="edit"> EDIT </button>
                <button v-else key="save">SAVE</button>
            </transition>
        </div>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data:{
                f:true
            }
        })
    </script>
</body>
过渡模式

新旧两个元素参与过渡的时候，新元素的进入和旧元素的离开会同时触发。可以使用transition提供的过渡模式mode，它有两个值，分别为in-out和out-in。前者为新元素进来后旧元素再离开，后者表示旧元素离开后新元素再进来。

多组件过渡

多组件过渡不需要使用key特性，使用动态组件即可。动态组件需要通过Vue中的<component>元素绑定is属性来实现。

3.3 列表过渡

对于列表过渡，需要使用v-for和transition-group组件来实现。

<transition-group>标签相当于给每个包裹的元素在外面加了一个transition标签。

列表的每一项都需要进行过渡，列表在循环时要给每一个列表项添加唯一的key属性值。

列表的进入和离开过渡

以下案例实现简单的列表过渡

<body>
    <div id="app">
        <!-- 绑定响应函数 -->
        <button @click="add">ADD</button>
        <button @click="remove">REMOVE</button>
        <!-- 定义过度类前缀 -->
        <!-- 修改tag,渲染出p标签 -->
        <transition-group name="list" tag="p">
            <!-- item赋给唯一的key属性 -->
            <span v-for="item in items" :key="item" class="list-item">
                {{item}}
            </span>
        </transition-group>
    </div>
    <script>
        var vm = new Vue({
            el:'#app',
            data:{
                items:[1,2,3,4,5],
                nextNum:6
            },
            methods:{
                randomIndex() {
                    return Math.floor(Math.random()*this.items.length)
                },
                add() {
                    this.items.splice(this.randomIndex(),0,this.nextNum++)
                },
                remove() {
                    this.items.splice(this.randomIndex(),1)
                }
            }
        })
    </script>
</body>

其中CSS样式为：

    <style>
        .list-item{
            display: inline-block;
            margin-right: 10px;
            background-color: red;
            border-radius: 50%;
            width: 25px;
            height: 25px;
            text-align: center;
            line-height: 25px;
            color: #fff;
        }
        .list-enter-active,.list-leave-active {
            transition: all 1s;
        }
        .list-enter,.list-leave-to {
            opacity: 0;
            transform: translateY(30px);
        }
    </style>
列表的排序过渡

通过v-move特性，在元素改变定位的过程中应用，可以通过name属性来自定义前缀，也可以通过move-class属性来设置自定义类名，和过渡类一致。

4.Vue路由

实现URL和组件之间的一一对应。更新视图而不更新请求页面（单页面应用SPA）。

4.1 vue-router

实现单页面前端路由时，提供了hash模式和history模式。默认为hash模式，使用URL的hash来模拟一个完整的URL，当URL改变时，页面不会重新加载。#就是hash符号，也成为锚点或者哈希符。hash值是指导浏览器动作的，对服务器没有影响。

基本使用如下。

<body>
    <div id="app">
        <div>
            <!-- 该标签实现跳转链接功能 -->
            <router-link to="/"> 首页 </router-link>
            <router-link to="/about"> 关于我们 </router-link>
        </div>
        <div>
            <!-- 占位符使用，将路由规则中匹配的组件进行展示 -->
            <router-view> </router-view>
        </div>
    </div>
    <!-- 组件模板 -->
    <template id="sy">
        <div><h1>首页</h1></div>
    </template>
    <template id="ab">
        <div><h1>关于我们</h1></div>
    </template>
    <script>
        // 路由表
        var routes=[
            {
                path:'/',
                component:{
                    template:'#sy'
                }
            },
            {
                path:'/about',
                component:{
                    template:'#ab'
                }
            }
        ]
        //通过全局路由构造函数构造一个路由器，其中配置路由表
        var router = new VueRouter({
            routes:routes
        })
        var vm = new Vue({
            el:'#app',
            //将路由规则对象注册到实例上
            router:router
        })
    </script>
</body>
query方式传参

参数值存放在query对象，通过user组件对象的$route属性访问当前路由，调用query获取相关参数信息。

使用query传参，参数会以查询字符串的形式显示在浏览器地址栏中。

<body>
    <div id="app">
        <router-link to="/user?id=10&name=admin"> Login </router-link>
        <router-view> </router-view>
    </div>
    <template id="tmp">
        <!-- 通过query获取参数对象 -->
        <h3>id:{{this.$route.query.id}} +  name:{{$route.query.name}}</h3>
    </template>
    <script>
        var user = {
            template:"#tmp",
            //钩子函数输出当前路由
            created() {
                console.log(this.$route)
            }
        }
        // 路由表对象
        var router = new VueRouter({
            routes:[
                {path:'/user', component:user}
            ]
        })
        var vm = new Vue({
            el:'#app',
            // 注册路由
            router:router
        })
    </script>
</body>
params方式传参

将参数放在路径中或者隐藏，如下例子。

<body>
    <div id="app">
        <!-- 此处会进行参数的解析 通过params进行参数的获取 -->
        <router-link to="/user/10/admin"> Login </router-link>
        <router-view> </router-view>
    </div>
    <template id="tmp">
        <!-- 插值表达式中省略了this -->
        <h3> id:{{$route.params.id}} name:{{$route.params.name}}</h3>
    </template>
    <script>
        var user = {
            template:'#tmp',
            created() {
                console.log(this.$route)
            }
        }
        var router = new VueRouter({
            routes:[
                // 在path中路径中以冒号的形式设置参数，传参id和name
                // 在link to中进行解析
                {path:'/user/:id/:name',component:user}
            ]
        })
        var vm = new Vue({
            el:'#app',
            router:router
        })
    </script>
</body>

4.2 嵌套路由

嵌套子路由通过属性children实现，children也是一组路由。在路由表中的某一路由对象中加入children属性即可。例如

<body>
    <div id="app">
        <ul>
            <router-link to="/about" tag="li">关于公司</router-link>
            <router-link to="/contact" tag="li">联系我们</router-link>
        </ul>
        <router-view> </router-view>
    </div>
    <template id="about-tmp">
        <div class="about-detail">
            <h1>北京xx科技有限公司简介</h1>
            <!-- 结构为：父路由地址/要去的子路由 -->
            <router-link to="/about/detail">公司简介</router-link> 
            <router-link to="/about/governance">公司治理</router-link>
            <router-view></router-view>
        </div>
    </template>
    <template id="contact-tmp">
        <div class="about-detail">
            <h1>联系我们</h1>
            <p>公司位于北京市海淀区中关村科技园内</p>
        </div>
    </template>

    <script>
        var about ={template:'#about-tmp'}
        var contact={template:'#contact-tmp'}
        //子路由对应的两个组件
        var detail = {
            template:'<p> xx 是全球领先的 </p>'
        }
        var governance = {
            template:'<p>公司坚持以客户为中心</p>'
        }
        var router = new VueRouter({
            //路由表
            routes:[
                //重定向路由 redirect
                {
                    path:'/',redirect:'/about'
                },
                //带有子路由的路由对象
                {
                    path:'/about',
                    component:about,
                    //子路由也是一组路由表
                    children:[
                        {
                            //这里的path不用加上/，否则默认都从根路径开始请求
                            path:'detail',component:detail
                        },
                        {
                            path:'governance',component:governance
                        }
                    ]
                },
                {
                    path:'/contact',component:contact
                }
            ]
        })
        var vm = new Vue({ el:'#app', router:router })
    </script>
</body>

4.3 命名路由

通过v-bind指令，绑定<router-link>标签的to属性，to属性中，name用于唯一标识某一条路由。在路由中，需通过name属性进行设置。

传入对象的话to前面要加冒号。

<body>
    <div id="app">
        <!-- 标签中绑定name属性，选定用哪一条路由，并传参数，该参数在路由对象通过params可获取 -->
        <router-link :to="{name:'user1',params:{id:123}}">Login</router-link>
        <router-view></router-view>
    </div>
    <script>
        var usercomponent ={
            template:'<h3> user component </h3>',
            created() {
                //打印当前路由的信息
                console.log(this.$route)
            }
        }
        var router = new VueRouter({
            //给路由中加入name属性，以标识一条路由
            routes:[
                {
                    // 使用冒号的形式匹配参数
                    path:'/user/:id',
                    // 为路由进行命名
                    name:'user1',
                    // 映射的组件
                    component:usercomponent
                }
            ]
        })
        // 挂载并注册路由
        var vm = new Vue({
            el:'#app',
            router:router
        })
    </script>
</body>

4.5 编程式导航

router.push()

可以导航到不同的URL地址。这个方法会向history栈添加一条新的记录。

router.replace()

用法和push类似，只是不会在history栈中添加新的记录。

router.go()

5.Vuex状态管理

Vuex对象store可以理解为一个容器，立马包含了应用中大部分的状态state，定义组件初始状态，相当于Vue实例中的data属性；mutations通过事件处理方法改变数组状态，最终将state状态反应到组件中，类似于Vue实例中的methods属性。

计数器案例
<div id="app">
    <button @click="increment"> + </button>
    <p>
        {{this.$store.state.count}}
    </p>
</div>
<script>
	const store = new Vuex.Store({
        state: {
            //定义初始数据count
            count:0
        },
        mutations: {
            //定义事件处理方法increase
            //接收参数为state初始状态，通过state获取到count的值并进行操作
            increase(state) {
                state.count++
            }
        }
    })
    var vm = new Vue({
        el:'#app',
        store,
        methods: {
            increment() {
                //显式的提交mutation，提交状态变更
                this.$store.commit('increase')
            }
        }
    })
</script>

7.组合式API

7.1 基本概念

组合式 API (Composition API) 是一系列 API 的集合，使我们可以使用函数而不是声明选项的方式书写 Vue 组件。它是一个概括性的术语，涵盖了以下方面的 API：

**响应式API：**例如 ref() 和 reactive()，使我们可以直接创建响应式状态、计算属性和侦听器。

**生命周期钩子：**例如 onMounted() 和 onUnmounted()，使我们可以在组件各个生命周期阶段添加逻辑。

**注入依赖：**例如 provide() 和 inject()，使我们可以在使用响应式 API 时，利用 Vue 的依赖注入系统。

<template>
  Count is: {{ state.count }}, doubleCount is: {{ state.doubleCount }}
  <button @click="add">加</button>
</template>
 
<script>
import { reactive, computed } from "vue";
 
export default {
  setup() {
    const state = reactive({
      count: 0,
      doubleCount: computed(() => state.count * 2),
    });
 
    function add() {
      state.count++;
    }
 
    return {
      state,
      add
    }
  }
}
</script>

7.2 setup组件

setup() 函数在组件创建 created() 之前执行。

setup() 函数接收两个参数 props 和 context。

第一个参数 props，它是响应式的，当传入新的 prop 时，它将被更新。

第二个参数 context 是一个普通的 JavaScript 对象，它是一个上下文对象，暴露了其它可能在 setup 中有用的值。

7.3 ref函数

ref() 函数可以根据给定的值来创建一个响应式的数据对象，返回值是一个对象，且只包含一个 .value 属性。

在 setup() 函数内，由 ref() 创建的响应式数据返回的是对象，所以需要用 .value 来访问。

7.4 reactive函数

作用：reactive是一个函数，接收一个普通的对象传入，把对象数据转化为响应式对象并返回
