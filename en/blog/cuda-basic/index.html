【学习笔记】CUDA编程实践

  • Liang Hangfeng
  • Cuda , 学习笔记
  • March 24, 2024

一、概念简述

1.1 GPU硬件简介

左侧为CPU结构，右侧为GPU结构。

GPU计算：由 CPU 调度、GPU 计算共同完成的异构计算任务。

CPU被称为主机Host，GPU被称为设备Device。

1.2 CUDA开发环境概述

CUDA提供了两层API，即CUDA驱动API和CUDA运行时API。

其中CUDA驱动API更加底层，为程序员提供了更为灵活的编程接口，CUDA运行时API是在CUDA驱动API的基础上构建的更为高级的API。性能几乎没有差别。

二、CUDA基础知识

2.1 CUDA程序结构

使用nvcc编译器，文件后缀为cu。

int main() {
	主机代码;
    核函数调用;
    主机代码;
    核函数调用;
    ......
    return 0;
}
__global__ void KernelFunc1() {
    
}
__global__ void KernelFunc2() {
    
}

主机代码主要负责 CPU 和 GPU 的内存管理、计算任务的分派，而核函数调用主要负责完成主要的计算工作。

  1. 核函数用前缀__global__来定义；
  2. 只能使用void返回类型，需要返回结构需要使用传参指针的方式；
  3. 核函数只能访问GPU内存；
  4. 核函数不能使用变长参数；
  5. 核函数不能使用静态变量；
  6. 核函数不能使用函数指针；
  7. 核函数具有异步性；
  8. __divice__ 修饰的函数称为设备函数，只能被核函数或是其它设备函数调用，只能在设备中执行;
  9. __host__ 修饰的函数称为主机函数，它就是主机端的普通 C++ 函数，在主机（CPU）中调用和执行，可以忽略。

2.2 内存分配

使用函数 cudaMalloc进行内存的分配。

函数原型如下：

cudaError_t cudaMalloc(void** d_ptr, unsigned int size);

第一个参数是一个指针的指针，即二级指针，因为这个函数需要在 GPU 显存上分配出一片空间，并且让 d_ptr 指向这个空间，由于 cudaMalloc 函数的返回值已经用于返回错误代码，因此需要传入一个二级指针，然后由 cudaMalloc 函数负责改变这个指针，使它指向刚分配出的那片设备内存。

【修改d_ptr的指针值为开辟的内存空间。例如cudaMalloc((void **)&d_x)，开始d_x为指向一个数据的指针，&d_x即为指向这个指针的指针，通过(void**)进行类型的强制转换，为一个二级指针，修改这个二级指针指向的值，则修改d_x，将其设置为开辟的内存空间的地址】

2.3 内存拷贝

CUDA核函数传入的参数必须是指向设备内存的指针，需要对设备内存进行初始化，采用拷贝的方式进行。使用函数cudaMemcpy，函数原型如下：

cudaError_t cudaMemcpy(void* d_ptr, void* h_ptr, unsigned int size, enum cudaMemcpyKind)

第一个参数是指向设备内存的指针，第二个参数是指向主机内存的指针，最后一个参数是 enum 类型的变量，用于指出数据传输的方向。

例如：

const int N = 1000000;
    const int M = sizeof(real) * N;
    real *h_x = (real *) malloc(M);
    real *h_y = (real *) malloc(M);
    real *h_z = (real *) malloc(M);
    for (int n = 0; n < N; n++) {
        h_x[n] = a;
        h_y[n] = b;
    }
    real *d_x, *d_y, *d_z;
    CHECK(cudaMalloc((void **) &d_x, M));
    CHECK(cudaMalloc((void **) &d_y, M));
    CHECK(cudaMalloc((void **) &d_z, M));
    CHECK(cudaMemcpy(d_x, h_x, M, cudaMemcpyHostToDevice));
    CHECK(cudaMemcpy(d_y, h_y, M, cudaMemcpyHostToDevice));

2.4 CUDA线程组织

核函数的调用语法如下：

kernel_function<<<grid_size, block_size>>>(parameters)

其中grid_size 和 block_size都可以是一个 dim3类型的结构体，或是一个 unsigned 类型的无符号整数变量。

前者为网格大小，后者是线程块的大小。一个核函数对应一个网格，每个网格具有多个线程块，每个线程块中有多个线程。线程是最基本的单位。

SIMD：核函数的调用逻辑遵循 Single Instruction Multiple Data（SIMD），即核函数的代码（Single Instruction）是对多份数据（Multiple Data）执行相同操作的一段指令。在调用核函数后，每个线程都会执行核函数中的代码。

内联变量gridDim存储网格的维度，即包含了多少线程块；blockDim存储线程块的维度，即包含了多少线程。

内敛变量blockIdx存储当前线程所在的线程块在网格中的编号；threadIdx存储当前线程在线程块当中的编号。

int tid = threadIdx.z * blockDim.x * blockDim.y + threadIdx.y * blockDim.x + threadIdx.x;
int bid = blockIdx.z * gridDim.x * gridDim.y + blockIdx.y * gridDim.x + blockIdx.x;

blockDim.x * blockDim.y相当于表示每个z上有几个线程。

线程块在x、y、z三个方向上允许的最大值分别为1024、1024、64，且乘积不能大于1024。

2.5 CUDA核函数的设计

需要做到：

  1. 确定线程与数据的对应（使用 CUDA 内建变量实现）；
  2. 对需要处理的数据执行操作；

例如：

void __global__ add(const double *x, const double *y, double *z, const int N) {
    const int n = blockDim.x * blockIdx.x + threadIdx.x;
    if (n < N) {  
        z[n] = x[n] + y[n];
    }
}

示例代码：

#include <math.h>
#include <stdio.h>

const double EPSILON = 1.0e-15;
const double a = 1.23;
const double b = 2.34;
const double c = 3.57;
void __global__ add(const double *x, const double *y, double *z);
void check(const double *z, const int N);

int main(void) {
    const int N = 100000000;
    const int M = sizeof(double) * N;
    double *h_x = (double*) malloc(M);
    double *h_y = (double*) malloc(M);
    double *h_z = (double*) malloc(M);

    for (int n = 0; n < N; ++n) {
        h_x[n] = a;
        h_y[n] = b;
    }

    double *d_x, *d_y, *d_z;
    cudaMalloc((void **)&d_x, M);
    cudaMalloc((void **)&d_y, M);
    cudaMalloc((void **)&d_z, M);
    cudaMemcpy(d_x, h_x, M, cudaMemcpyHostToDevice);
    cudaMemcpy(d_y, h_y, M, cudaMemcpyHostToDevice);

    const int block_size = 128;
    const int grid_size = N / block_size;
    add<<<grid_size, block_size>>>(d_x, d_y, d_z);

    cudaMemcpy(h_z, d_z, M, cudaMemcpyDeviceToHost);
    check(h_z, N);

    free(h_x);
    free(h_y);
    free(h_z);
    cudaFree(d_x);
    cudaFree(d_y);
    cudaFree(d_z);
    return 0;
}

void __global__ add(const double *x, const double *y, double *z, const int N) {
    const int n = blockDim.x * blockIdx.x + threadIdx.x;
    if (n < N) {
        z[n] = x[n] + y[n];
    }
}

void check(const double *z, const int N) {
    bool has_error = false;
    for (int n = 0; n < N; ++n) {
        if (fabs(z[n] - c) > EPSILON) {
            has_error = true;
        }
    }
    printf("%s\n", has_error ? "Has errors" : "No errors");
}

2.6 编译CUDA程序

编译CUDA程序需要使用Nvidia官方提供的编译器nvcc。nvcc会先将所有源代码先分离成主机代码和设备代码，主机代码完整支持C++语法，设备代码只部分支持C++语法。nvcc先将设备代码编译为PTX（parallel thread execution）伪汇编代码，再将PTX代码编译为二进制的cubin目标代码。

2.7 错误检测

对于返回cudaError_t的CUDA API函数的错误检测，我们可以采用以下宏函数实现：

#define CHECK(call)                     
do                                  \
{                                   \
    const cudaError_t error_code = call;                \
    if (error_code != cudaSuccess)                  \
    {                                   \
    printf("CUDA ERROR:\n");                    \
        printf("    FILE:   %s\n", __FILE__);           \
        printf("    LINE:   %d\n", __LINE__);           \
        printf("    ERROR CODE: %d\n", error_code);         \
        printf("    ERROR TEXT: %s\n", cudaGetErrorString(error_code)); \
    exit(1);                            \
    }                                   \
} while(0)

对于无返回值的CUDA核函数的错误检测，可以在调用核函数后立刻调用 cudaGetLastError 和 cudaDeviceSynchronize 两个函数并使用如上方法解析它们的 cudaError_t 返回值，来判断刚刚运行的核函数发生了什么错误，具体如下：

kernel_function(parameters);
CHECK(cudaGetLastError());
CHECK(cudaDeviceSynchronize());

cudaGetLastError()函数用于捕捉第二个语句之前的最后一个错误，即cudaDeviceSynchronize()之前的错误，该语句用于同步主机和设备。

工具cuda-memcheck可以直接对可执行文件进行内存检查。

三、CUDA加速关键

3.1 CUDA事件计时

定义两个CUDA事件类型cudaEvent_t的变量start和stop，使用cudaEventCreate()函数进行初始化。

将start传入cudaEventRecord，记录一个代表开始的事件。对于cudaEventQuery()为刷新被提交至软件队列的操作，这些操作不会被直接提交至GPU。

使用cudaEventRecord表示一个结束的事件，使用cudaEventSynchronize()函数让主机等待stop事件被记录完毕，计算时间输出即可。

cudaEvent_t start, stop;
CHECK(cudaEventCreate(&start));
CHECK(cudaEventCreate(&stop));
CHECK(cudaEventRecord(start));
cudaEventQuery(start);

add<<<grid_size, block_size>>>(d_x, d_y, d_z);

CHECK(cudaEventRecord(stop));
CHECK(cudaEventSynchronize(stop));
float elapsed_time;
CHECK(cudaEventElapsedTime(&elapsed_time, start, stop));
printf("Time = %g ms. \n", elapsed_time);
有关精度的宏定义
#ifdef USE_DP
    typedef double real;
    const real EPSILON = 1.0e-15;
#else
    typedef float real;
    const real EPSILON = 1.0e-6f;
#endif

可以在编译是使用-DUSE_DP参数选择使用双精度运算。

nvprof工具

使用命令nvprof exe可以对CUDA程序进行性能剖析。

3.2 数据传输比例

GPU计算核心和设备内存之间数据传输的带宽要远高于GPU和CPU之间数据传输的带宽，也就是说，使用cudaMemcpy将数据从CPU转移到GPU的操作是非常费时的。

要想获得可观的GPU加速，需要缩短数据传输所花时间的比例。

如果实际核函数中计算任务很重，那么CPU和GPU之间数据传输的时间就可以忽略不计，此时加速效果就会非常明显。

3.3 核函数算术强度和访存时间

一个计算问题的算术强度是指其中算术操作的工作量（时间）与必要的内存操作的工作量（时间）之比。算术强度越高则加速效果越好。

  1. 尽可能减少访存操作的次数
  2. 通过共享内存、寄存器内存等CUDA内存结构的合理使用来提高访存操作的次数

3.4 SM驻留线程数和并行规模

SM（Stream Multiprosessor）即流多处理器，一个GPU中包含众多SM（一般每个SM中最多能够驻留的线程数目为1024或者2048，视架构而定）。要使一个CUDA程序达到最优性能，就需要提高每个流处理器能够驻留的线程数量。

并行规模可以定义为一个GPU中同时驻留的线程数目，我们可以通过调用核函数时指定gridDim和blockDim来确定并行规模。可以这样理解：

  • 并行规模是从整个GPU处理器的角度看待驻留线程的数量。
  • SM中驻留线程的数量则是从更微观角度看待。

影响SM中驻留线程数主要有两个因素：

  • 并行规模。当并行规模不够大时，实际分配每个SM的线程数量太少，远远不能达到它能够处理的理论最大线程数，这时SM 的占有率很低，不能激发 GPU 的最大性能。
  • 资源限制。每个 SM 都具有一定数量的寄存器、共享内存等资源，当我们在核函数中使用这些资源超过一定量时（单个线程使用量超标），SM 就不得不减少其内部驻留的线程数。

SM 总资源 = 单个线程占有资源数 * SM中线程数量

对于并行规模，这是根据具体问题确定的，这也是为什么更大规模的问题更具有利用 GPU 进行并行计算的价值。而对于资源限制，我们将在本章节中详细介绍完 CUDA 内存结构后单开一个话题讨论。
