一. Linux操作系统介绍

操作系统

系统软件：管理计算机资源/硬件，并为应用软件提供一个统一的平台。

应用软件：在系统软件的基础上实现用户所需要的各种功能。

操作系统（Operating System，OS） 是最基本的系统软件，负责对电脑硬件直接控制及管理，包括处理器、存储、设备和作业等。

常见操作系统

Unix：1969 贝尔实验室

MS-DOS：为IBM个人电脑开发，单用户单任务，1981

Mac OS：1984

Windows：1985

Linux：1991，类 Unix 操作系统

FreeBSD：Intel平台，1993

UNIX

服务器领域，大多网络环节，后用C语言改写，版本：AIX (IBM)、 HP-UX (HP)、 Solaris(SUN) 、 IRIX(SGI)

不能用于 Intel 平台

以 UNIX 为基础形成的开放系统标准 (POSIX) 也是迄今为止唯一的操作系统标准

GNU/Linux介绍

软件按其提供方式分类：商业软件、共享软件、自由软件

GNU-GNU is Not Unix，一个完全基于自由软件的软件体系计划

Linux免费使用，自由传播，高效性和灵活性 ，有多任务、多用户的能力，支持多种硬件平台

版本众多： RHEL、 CentOS、 Fedora、 Ubuntu、

1987：简化的类 UNIX 系统 (miniUNIX， Minix) 可用于386、 486或奔腾处理器的个人计算机上，并且具有 Unix 操作系统的全部功能

1990：Linux 0.0.1 版本，但它只有操作系统内核，必须在有 Minix 的机器上编译后才能用

1991：Linux 0.0.2 版本，可以独立运行 bash 和 gcc （Linux第一个内核）

Linux系统结构

内核、 Shell、文件系统、应用程序

内核：运行程序和管理各种硬件设备的核心程序

Shell：命令语言解释器，是用户与内核进行交互操作的接口

文件系统：文件存放在存储设备上的组织方法，如 ext4、 vfs、 nfs 等

应用软件：Linux 发行版都有一整套的应用程序集

Linux的主要特点

开放性：系统遵循世界标准规范，完全符合 POSIX 标准。

多用户：系统资源可以被不同用户使用，每个用户对自己的资源有特定的权限，互不影响。

多任务：计算机可同时执行多个程序，且各个程序的运行互相独立。

良好的用户界面：Linux 同时向用户提供了字符界面和图形界面，给用户呈现一个直观、易操作、交互性强的友好的图形化界面。

设备独立性：Linux 把所有外部设备统一当作文件来看待，任何用户都可以象使用文件一样，操纵、使用这些设备。

丰富的网络功能：完善的内置网络是Linux一大特点。

可靠的安全系统 ：Linux 采取多种安全技术措施，为网络多用户环境中的用户提供了必要的安全保障。

良好的可移植性：Linux 是一种可移植的操作系统，能够在从微型计算机到大型计算机的任何环境中和任何平台上运行。

常见的发行版

RedHat Linux 、Fedora 、Debian 、Mandrake 、SuSE、Ubuntu、Red Flag Linux、TurboLinux 、蓝点 Linux

二、Linux基本操作与管理

Linux基础

术语

命令：给计算机的指令，通常使用键盘输入， Linux 区分大小写

命令行：在终端里输入命令的地方

Shell：一种命令解释器，提供用户和操作系统之间的交互接口； 当用户输入命令后， shell 进行解释，然后送往操作系统（内核）执行； shell 可以执行 Linux 系统内部命令，也可以执行应用程序；也可以利用 shell 编程，执行复杂的命令程序

超级用户/根用户/管理员（root）：具有系统最高权限，对整个系统有完全的访问权；只有超级用户才能更改系统的设置

Linux用户管理

user：有一个用户ID号uid，输入id可显示信息，分配到一个组，每个组有ID号gid

用户登录后，会进入系统指定的专属目录，即用户的主目录（home） ，主目录的名字通常与用户帐号相同

用户对自己主目录下的文件拥有所有权，可以进行相关操作

在创建新用户时，管理员会给每个用户建立一个主目录

Linux命令

所有的命令从标准输入接受输入，输出结果显示在标准输出，而错误信息则显示在标准错误输出设备

命令在正常执行结果后返回一个 0 值，否则返回一个非零值(在shell中可用变量 $? 查看： echo $?)

修改密码（口令）

passwd

yppasswd：修改NIS口令，网络用户需要使用该命令来修改密码

帮助信息

帮助信息：man 指令；

更多信息：info 指令；

Linux文件系统结构

目录树结构

从一个根目录“ / ”开始，根目录下可以有任意多个文件和子目录，子目录中又可以有任意多个文件和子目录，形成一个树状结构

当前工作目录

用户登录后进入的目录通常是自己的主目录

pwd 命令查看用户的当前目录

cd命令切换牡蛎

“.” 表示当前目录

“..” 表示当前目录的上一级目录（父目录）

“~” 表示用户主目录的绝对路径名

“-” 表示用 cd 命令切换目录前所在的目录

路径

任何不以 / 和 ~ 开始的路径均为相对路径

例如：当前工作目录为/home/jypan/Teaching/

则该目录的子目录 ParaComp/ 中的文件 lect01.pptx 的绝对路为/home/jypan/Teaching/ParaComp/

相对路径为./ParaComp/

文件访问权限

文件类型：普通文件、目录、符号链接、设备文件等

三种限制访问权限的方式：只允许用户自己访问、允许与自己同组的用户访问、允许系统内的所有用户访问（三种不同类型的用户：所有者、同组用户、其它用户）

一个文件或目录可以有读、写和执行的权限。当一个文件或目录被创建时，系统会自动赋予文件所有者的读和写的权限，这样就允许文件所有者能够阅读和修改该文件。

使用 ls -l 查看文件和目录的属性

第一个字段为文件类型：-普通文件 d目录 l链接文件

9个字符表示权限，3个一组为三套控制权限，分别为所有者、同组成员、其它用户的访问权限

r：read（文件阅读，目录浏览）

w：write（文件修改，目录移动修改目录内文件）

x：execute执行（文件执行，目录进入）

-：不具备权限，取消该权限

用户主目录的默认权限通常为： rwx——

修改访问权限

chmod

文字设定法（使用字母和操作符表达式来修改或设定文件的访问权限 ）：

chmod [操作对象] [+|-|=] [权限] 文件或目录列表

操作对象：u所有者 g同组用户 o其它用户 a所有用户 默认a

操作符：+添加 -取消 =赋予

权限：rwx

文件或目录列表：可以单个或多个文件/目录，支持通配符

例如：chmod o=rx mydata/

数字设定法

每一组权限转换为数字 如rwx为7 –x为1

chmod 700 filename # 设定权限为： rwx——

进程管理

正在运行的程序叫做进程（process），程序只有被系统载入内存并运行后才能称为进程。

分配进程号（process ID），每个进程的进程号唯一

Linux 是多用户多进程操作系统

ps查看当前进程

中止进程 ctrl+c kill killall

前台与后台

前台进程：指一个程序控制着标准输入/输出，在程序运行时， shell 被暂时挂起，直到该程序运行结束后，才退回到 shell。在这个过程中，用户不能再执行其它程序。

后台进程：用户不必等待程序运行结束就可以执行其它程序。

在一个终端里只能同时存在一个前台任务，但可以有多个后台任务。

运行后台进程：加上&

如果程序已经在前台运行，需要将其改为后台运行，可以先安组合键ctrl+z，将任务挂起，然后利用 bg 命令将该程序转为后台运行

若要将一个后台进程转到前台运行，可以使用 fg 命令

u 增加显示用户名，进程起始时间， CPU 和内存占用百分比等信息

a 显示终端机下用户执行的进程，包含其它用户

r 显示正在运行的进程

-o 指定输出文件名 缺省为xxx.out

-O1 -O2 -O3 优化开关

GCC编译工具链

GCC 编译⼯具链（toolchain） 是指以 GCC 编译器为核⼼的⼀整套⼯具，⽤于把源代码转化成可执⾏应⽤程序。

包含：

GCC 编译器 : ⽤于完成预处理和编译过程，例如把 C 代码转换成汇编代码。

glibc：包含了主要的 C 语⾔标准函数库， C 语⾔中常常使⽤的打印函数 printf、 malloc 函数就在 glibc 库中。

Binutils ：除GCC编译器外的⼀系列⼩⼯具包括了链接器ld， 汇编器as、 ⽬标⽂件格式查看器readelf等。

编译器就是把⾼级语⾔翻译成机器语⾔的⼀个⼯具。编译过程⼀般可以分为6步：词法分析、语法分析、语义分析、源代码优化、⽬标代码⽣成与优化。

GCC编译过程

gcc hello.c -o hello
./hello

事实上，上述过程可以分解为4个步骤，分别是预处理、编译、汇编和链接

(hello.c stdio.h)→[预处理]→ hello.i →[编译]→ hello.s →[汇编]→ hello.o

(hello.o libc.a) →[链接]→ hello

-E只预处理

-S只编译

-C只汇编

gcc -E hello.c -o hello.i
gcc -S hello.i -o hello.s
gcc -c hello.s -o hello.o

预处理：合并头文件内容

编译：产生对应的汇编代码文件

汇编：将汇编代码翻译成机器代码，一般表现为ELF目标文件（OBJ文件）

链接：链接通俗讲就是将上步⽣成的OBJ⽂件和系统库的OBJ⽂件、库⽂件链接起来，最终⽣成可以在特定平台运⾏的可执⾏⽂件。可以在编译时由静态编译器来完成 ，也可以在加载时和运⾏时由动态链接器来完成

Make与Makefile

make⼯具：它可以帮助我们找出项⽬⾥⾯修改变更过的⽂件，并根据依赖关系，找出受修改影响的其他相关⽂件，然后对这些⽂件按照规则进⾏单独的编译，这样⼀来，就能避免重新编译项⽬的所有的⽂件。

Makefile⽂件：上⾯提到的规则、依赖关系主要是定义在这个Makefile⽂件中的，我们在其中合理地定义好⽂件的依赖关系之后，make⼯具就能精准地进⾏编译⼯作。

Makefile

管理项⽬⽂件间的依赖关系

源码编译过程
  1. 配置configure
  2. 确定标准库和头文件的位置
  3. 确定依赖关系
  4. 头文件的预编译
  5. 预处理
  6. 编译
  7. 链接 （make命令从4到7）【通过拷⻉，将外部函数库添加到可执⾏⽂件的⽅式，叫做静态连接（static linking）， 其他还有动态连接（dynamic linking）。】
  8. 安装
  9. 操作系统连接（make install 命令8-9）
  10. 生成安装包（更方便）
动态连接和静态连接

静态连接就是把外部函数库，拷⻉到可执⾏⽂件中。这样做的好处是，适⽤范围⽐较⼴，不⽤担⼼⽤户机器缺少某个库⽂件；缺点是安装包会⽐较⼤，⽽且多个应⽤程序之间，⽆法共享库⽂件。

动态连接的做法正好相反，外部函数库不进⼊安装包，只在运⾏时动态引⽤。好处是安装包会⽐较⼩，多个应⽤程序可以共享库⽂件；缺点是⽤户必须事先安装好库⽂件，⽽且版本和安装位置都必须符合要求求，否则就不能正常运⾏

链接过程：

程序源代码编译后的机器指令被放到代码段（.text）

全局变量和局部变量被放到数据段（.data）

未初始化的全局变量和局部变量被放到.bss段

静态链接的链接器链接过程：

  1. 空间与地址分配

扫描所有的输⼊⽬标⽂件，获得它们的各个段的⻓度、属性和位置，并且将输⼊⽬标⽂件中的符号表中所有的符号定义和符号引⽤收集起来，统⼀放到⼀个全局符号表。这⼀步中，链接器将能够获得所有输⼊⽬标⽂件的段⻓度，并且将它们合并，计算出输出⽂件中各个段合并后的⻓度与位置，并建⽴映射关系

  1. 符号解析与重定位

使⽤上⾯第⼀步收集到的所有信息，读取输⼊⽂件中断的数据、重定位信息，并进⾏符号解析与重定位、调整代码中的地址

五、Shell与Makefile

Shell介绍

Shell 是一个用 C 语言编写的程序，它是用户使用 Linux 的桥梁。Shell 既是一种命令语言，又是一种程序设计语言。

Shell 是指一种应用程序，这个应用程序提供了一个界面，用户通过这个界面访问操作系统内核的服务。

Shell环境

脚本解释器：Bash（Bourne Again Shell）也是大多数Linux 系统默认的 Shell。

Shell脚本编程

脚本实例
#!/bin/bash
echo "helloworld"

#! 是一个约定的标记，它告诉系统这个脚本需要什么解释器来执行，即使用哪一种 Shell。

sh变量

变量名和等号之间不能有空格，变量名不加美元符

只包含字母、数字和下划线；不能以数字开头；避免使用特殊符号

语句给变量赋值：

for file in `ls / etc`
for file in $(ls /etc)

使用变量：(变量名外面的花括号是可选的)

name="hello"
echo $name
echo ${name}

重新定义（第二次赋值不加上美元符，使用变量采用美元符）

name="tom"
echo $name
name="ali"
echo $name

只读变量：readonly 变量名

删除变量：unset 变量名，unset 命令不能删除只读变量

Makefile介绍

makefile来告诉make命令如何编译和链接多个文件

target...: prerequisites...
	recipe
	...
	...

如果要删除可执行文件和所有的中间目标文件，执行 make clean

案例
edit :main.o kbd.o command.o display.o\
		insert.o search.o files.o utils.o
	cc -o edit main.o kbd.o command.o display.o\
		insert.o search.o files.o utils.o
main.o :main.c defs.h
	cc -c main.c
kbd.o : kbd.c defs.h command.h
	cc -c kbd.c 
command.o:command.c defs.h command.h
	cc -c command.c
display.o :display.c defs.h buffer.h
	cc -c display.c
insert.o :insert.c defs.h buffer.h
	cc -c insert.c
search.o :search.c defs.h buffer.h
	cc -c search.c
files.o :files.c defs.h buffer.h command.h
	cc -c files.c
utils.o : utils.c defs.h
	cc -c utils.c
clean :
	rm edit main.o kbd.o command.o display.oinsert.o search.o files.o utils.o

如果prerequisites文件的日期要比targets文件的日期要新，或者target不存在的话，那么，make就会执行后续定义的命令

clean 不是一个文件，它只不过是一个动作名字。其冒号后什么也没有，make不会自动去找它的依赖性，也就不会自动执行其后所定义的命令。要执行其后的命令，就要在make命令后明显得指出这个label的名字

六、Linux系统移植

Linux系统组成

在移植Linux之前我们需要先移植⼀个bootloader代码，这个bootloader代码⽤于启动 Linux 内核，bootloader有很多，常⽤的就是U-Boot。

还需要再移植⼀个根⽂件系统(rootfs)， 根⽂件系统⾥⾯包含了⼀些最常⽤的命令和⽂件。

三要素

U-Boot、 Linux kernel 和 rootfs 这三者⼀起构成了⼀个完整的 Linux 系统。

内核编译结果

内核编译完成以后，将⽣成⼏个重要的⽂件。它们是vmlinux、 vmlinuz和System. map。

  1. vmlinux是在内核源码顶层⽬录⽣成的内核映像。

  2. System.map是⼀个特定内核的内核符号表，它包含内核全局变量和函数的地址信息。

  3. zImage是⼀个合成的内核镜像，可以由引导加载程序加载并启动。 他包含了⼀段启动加载程序和把原始内核主体压缩之后的⼀段数据。

  4. uImage是把zImage加上64字节的头信息后产⽣的。U-Boot在加载uImage镜像时会检查这64字节的信息，确定镜像是否正确，确定加载地址和⼊⼝地址等，从⽽能正确引导zImage镜像。

Linux设备树

组成

设备树源码DTS，设备树源码文件DTSI，设备树编译工具DTC，二进制格式设备树DTB。

过程

在Uboot启动内核时，会先读取DTB到内存，然后再启动内核，内核可以从设备树中获取硬件信息。

基本数据格式

在Linux中，设备树⽂件的类型有“.dts”， “.dtsi”和“.dtb”。其中“.dtsi”是被包含的设备树源⽂件，类似于C语⾔中的头⽂件； “.dts”是设备树源⽂件，可以包含其他“.dtsi”⽂件，由dtc编译⽣成“.dtb”⽂件。

设备树是⼀个包含节点和属性的简单树状结构。

/ {
    node1 {
        a-string-property = "A string";
        a-string-list-property = "first string", "second string";
        a-byte-data-property = [0x01 0x23 0x34 0x56];
        child-node1 {
            first-child-property;
            second-child-property = <1>;
            a-string-property = "Hello, world";
        };
        child-node2 {
        };
    };
    node2 {
        an-empty-property;
        a-cell-property = <1>; /* each number (cell) is a uint32 */
        child-node1 {
        };
    };
};

上⾯的设备树例⼦包含了下⾯的内容：

  1. ⼀个单独的根节点： “/”，
  2. 两个⼦节点： “node1”和“node2”，
  3. 两个 node1 的⼦节点： “child-node1”和“child-node2”

移植概念

所谓移植就是把程序代码从⼀种运⾏环境转移到另外⼀种运⾏环境。对于内核移植来说，主要是从⼀种硬件平台转移到另外⼀种硬件平台上运⾏。

所涉及到的Linux内核移植⼯作也就分为了3各层次。 ⾸先，内核代码中要添加相应ARM核的⽀持；其次，内核代码中要添加相应SoC的⽀持；最后就是内核代码中要添加相应硬件单板的⽀持。

简要步骤
# 编译 uImage和 设备树文件
make ARCH=arm exynos_defconfig
make ARCH=arm CROSS_COMPILE=arm-linux- uImage -j2
make ARCH=arm CROSS_COMPILE=arm-linux- dtbs
# 拷贝到tftp服务器
sudo cp arch/arm/boot/uImage /var/lib/tftpboot/
$ sudo cp arch/arm/boot/dts/exynos4412-origen.dtb /var/lib/tftpboot/

在开发板终端执行：

tftp 42000000 exynos4412-origen.dtb
bootm 41000000 - 42000000

Linux内核已经启动起来了，但是最终的Linux操作系统还没有运⾏起来。原因是内核启动的最后阶段挂载根⽂件系统还没有成功

根文件系统

根⽂件系统（rootfs） 是linux在初始化时加载的第⼀个⽂件系统，

根⽂件系统是在内核启动时直接挂载在根⽬录（/）下的⼀个⽂件系统。他包含了Linux系统运⾏所必须的⼀些⼯具、库、脚本、配置⽂件和设备⽂件等。没有挂载根⽂件系统的⼀个单纯的Linux内核仅仅只是⼀个内核，没有实际的⽤处。

根文件系统目录结构

⽂件系统是操作系统⽤于明确磁盘或分区上⽂件的组织和访问的⽅法。

⽽Linux的⽂件系统组织和DOS操作系统不同，它的⽂件系统是⼀个整体，所有的⽂件系统结合在⼀个完整的统⼀的树形⽬录结构之中。 ⽬录是树的枝⼲， 这些⽬录可能会包含其他⽬录，或是其他⽬录的“⽗⽬录”， ⽬录树的顶端是⼀个单独的根⽬录，⽤“/”表示。其他磁盘或磁盘分区都挂载在根⽬录的某⼀个⼦⽬录下⾯。

FHS目录结构

Linux基⾦会发布了⽂件系统层次结构标准（ Filesystem Hierarchy Standard ， FHS ）。

bin dev etc home lib mnt proc root sbin sys tmp usr var
  1. /bin：包含⼆进制（binary）⽂件形式的可执⾏程序 ，ls、mkdir都是该在目录下的可执行文件
  2. /dev：在/dev⽬录下是⼀些称为设备⽂件的特殊⽂件，⽤于访问系统资源或设备（在Linux下，所有设备都被抽象成了⽂件，有了这些⽂件，⽤户可以像访问普通⽂件⼀样⽅便地访问系统中的设备。）
  3. /etc：Linux的很多系统配置⽂件就在该⽬录下，例如系统初始化脚本⽂件/etc/rc等
  4. /lib：必要的共享库和内核模块
  5. /mnt：该⽬录⽤来为其他⽂件系统提供安装点
  6. /proc：/proc⽂件系统并不保存在系统的硬盘中，操作系统在内存中创建这⼀⽂件系统⽬录，是虚拟的⽬录， 即系统内存的映射， 其中包含⼀些和系统相关的信息
  7. /root：root⽤户主⽬录
  8. /sbin：与bin⽬录类似， 存放系统的可执⾏⽂件、命令，通常只有root用户才有运行的权限
  9. /sys：sysfs伪⽂件系统标准挂载点，类似于proc⽬录，能查看和设定内核参数，但主要⽤于设备管理⽤。
  10. /tmp：公⽤的临时⽂件存储点
  11. /usr：/usr是最庞⼤的⽬录，该⽬录中包含了⼀般不需要修改的命令程序⽂件、程序库、⼿册和其他⽂档等。 Linux内核的源代码就放在/usr/src/⾥。
  12. /var：该⽬录中包含经常变化的⽂件，例如打印机、邮件、新闻等的脱机⽬录、⽇志⽂件以及临时⽂件等。
Linux内核启动

存在根⽂件系统是Linux启动时的必要条件

  1. ⾸先内核会初始化⼀个基于内存的⽂件系统，如initramfs， initrd等
  2. 然后以只读的⽅式去加载根⽂件系统（load rootfs）， 读取并且运⾏/sbin/init初始化⽂件，根据/etc/inittab配置⽂件完成系统的初始化⼯作（提示： /sbin/init是⼀个⼆进制可执⾏⽂件，为系统的初始化程序，⽽/etc/inittab是它的配置⽂件）【init初始化进程】
  3. 在初始化的过程中，还会以读写的⽅式᯿新挂载根⽂件系统，在系统启动后， 根⽂件系统就可⽤于存储数据了
根文件系统制作

常⻅的根⽂件系统制作⼯具有 buildroot、 Ubuntu、 Debian、yocto、 busybox

busybox优点：体积小、支持常用命令、常用于定制特定功能的小系统，缺点：功能不全，无包管理工具

Busybox的配置分为三⼤类，第⼀类是Busybox的基本配置，包括编译、调试安装的选项；第⼆类是⼯具集的选择，可以选择Busybox最终⽀持的⼯具；第三类是配置⽂件的加载和保存的路径设置。

其中bin、 sbin和usr⽬录下是各种⼯具的软连接，最终都指向与bin⽬录下的busybox可执⾏⽂件，⽬前这种配置下该⽂件的⼤⼩只有900多K字节。

Linux内核启动过程

Linux内核启动就是引导内核镜像启动的过程。

典型的内核镜像是uImage，包含 64字节的U-Boot头，启动加载程序和压缩后的内核主体 3部分。

启动过程中最主要任务的就是解压和启动内核主体。

基本内核移植过程回顾
  1. 获取内核源码
  2. 解压内核源码
  3. 进行完整性检查
  4. 给内核源码打补丁
  5. 清除内核
  6. 对内核进行默认配置
  7. 编译内核源码
  8. 生成设备树文件
  9. 拷贝内核和设备树文到特定目录下
  10. 启动内核（过程看指令）
